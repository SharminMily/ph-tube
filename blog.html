1. Discuss the scope of var, let, and const

var, let, and const are three different ways to declare variables in JavaScript, each with its own scope.
Here's an overview of the scope of each:

1. var:

  • Variables declared with var have function scope or global scope, but they are not block-scoped.
  • If declared outside any function, they become global variables accessible from anywhere in the script
  • Hoisting: var variables are hoisted to the top of their function or global scope, which means you can access them before they are actually declared in the code.
    • OR

    2.let:

  • let variables are not hoisted to the top of the block, so you can only access them after they have been declared.
  • If declared within a function, they are only accessible within that function.
  • If declared outside a function, they have global scope and can be accessed from anywhere in the script.
    • OR

    3. const:

  • Variables declared with const are also block-scoped like let.
  • Like let, const variables are not hoisted and must be declared before they can be used.
  • const variables are often used to represent values that are not expected to change, improving code readability and preventing accidental reassignments.
  • Variables declared with const are also block-scoped like let.

    null and undefined are two distinct values in JavaScript that represent the absence of a value.
    Let's explore the use cases of both:

    1. null:

  • The null value is used to represent the intentional absence of any object value.
  • It is often used to indicate that a variable should be empty or have no value.
  • Use cases include scenarios where you want to reset a variable or indicate that an object property has no value.

    • Example:

    3. What do you mean by REST API?

    REST, or Representational State Transfer, is an architectural style for designing networked applications, particularly web services. A REST API (Application Programming Interface) is a set of guidelines and conventions that allows different software applications to communicate with each other over the internet using the principles of REST.
    Key characteristics of a REST API include:

  • Resources: REST APIs model data and functionality as resources, which can be thought of as objects or entities. Each resource is identified by a unique URL (Uniform Resource Locator).
  • Resources: REST APIs model data and functionality as resources, which can be thought of as objects or entities. Each resource is identified by a unique URL (Uniform Resource Locator).GET,POST,PUT,DELETE
  • Stateless: Each client request to a REST API should contain all the information needed to process the request. The server doesn't maintain any client state between requests.
  • Uniform Interface: REST APIs have a consistent and uniform way of interacting with resources
  • Representation: Resources in a REST API are represented in different formats, such as JSON (JavaScript Object Notation) or XML. Clients and servers can negotiate the appropriate representation format based on the Accept and Content-Type headers.
  • Hypermedia as the Engine of Application State (HATEOAS): This principle encourages including hyperlinks (URLs) in API responses, allowing clients to discover and navigate to related resources without needing prior knowledge of the API structure.
